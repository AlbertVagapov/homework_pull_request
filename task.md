# Инструкция по работе с Git репозиторием 

## Для начало работы: 
>git init 

Если имя пользователя и почта ещё не были заданы:

>config --global user.name

>config --global user.email

## Работа с файлами:
Для того чтобы добавить файл, нужно:
1. >add file_name

добавляем файл с именем file_name для отслеживания 
2. >git commit -m "some message"

* добавляем текущий изменения в репозиторий и подписываем их с помощью тега -m 
* если добавить тег -а, то к комиту добавятся все измененные файлы, которые уже отслеживались  

Чтобы отслеживать состояние репозитория:

1. > git status 

2. > git log

3. > get diff

## Работа с комитами
Для того чтобы перейти к определенному коммиту можно использовать команду:

>git checkout code_commit 

code_commit - код коммита к которомц хотим перейти, его можно посмотреть в git log 

Чтобы вернуться к прошлому состоянию:
>git checkout master

Чтобы вызвать справку необходимо ввести:

>git --help 

## Ветки в Git
Чтобы посмотреть все ветки 

>git branch

Для создания новой ветки с именем branch_name

>git branch branch_name

Переместиться к ветке с имнеим branch_name:

>git checkout branch_name

## Слияние веток и решение конфликтов
Чтобы слить информацию из ветки branch_name в текущую:

>git merge branch_info

В случае конфликта нужно удалить все лишние строки и оставить ту часть, которая нам нужна. Если неодходимо, то можно отредактировать её. 

## Удаление веток
Для того чтобы удалить ветку с именем branch_name:

>git branch -d branch_name

Удаление с игнорированием ошибок:

>git branch -D branch_name

Чтобы вызвать справку необходимо ввести:

>git --help 

Примеры:

>git add --help

>git branch --help

>git status --help

Для привязки локального репозитория, который был создан ранее чем в github, необходимо ввести следующие команды:

>git remote add origin (ssh_link)

>git push --set-upstream origin master

Для генерации ключа ввести команду ниже. Далее скопировать ключ с папки и вписать его в https://github.com/settings/keys

>ssh-keygen -t rsa -b 4096 -C "e-mail"
